\noindent This document is the source of truth for the reusable simulator code under \texttt{cqed\_sim/}. It records the active physics conventions, the protocol-facing abstractions that now sit on top of the simulator, the cleaned package boundary versus \texttt{examples/}, and the tests that directly back the main conventions.

\section{Overview of cqed\_sim}
\texttt{cqed\_sim/} now contains only reusable simulator code:
\begin{itemize}[leftmargin=1.5em]
  \item \texttt{core}: Hilbert-space conventions, frames, the two-mode dispersive transmon-storage model, the three-mode storage-transmon-readout model, ideal gates, and manifold-frequency helpers.
  \item \texttt{pulses}, \texttt{sequence}, \texttt{sim}: pulse objects, envelopes, channel compilation, hardware distortion, the solver path, collapse operators, and extractors.
  \item \texttt{experiment}: reusable state preparation, qubit measurement, and a lightweight protocol wrapper built on the same compile-and-simulate path as the low-level runtime.
  \item \texttt{calibration}, \texttt{observables}, \texttt{operators}, \texttt{tomo}, \texttt{io}, \texttt{plotting}, \texttt{unitary\_synthesis}: reusable helpers that still belong to the library surface.
\end{itemize}
Study, workflow, paper-reproduction, audit, and smoke-test code remains under \texttt{examples/}. The alignment note for the experiment-facing cleanup is kept in \texttt{physics\_and\_conventions/experimental\_protocol\_alignment\_note.md}.

\section{Experimental Protocol Layer}
The main experiment-facing additions are implemented in:
\begin{itemize}[leftmargin=1.5em]
  \item \texttt{cqed\_sim/experiment/state\_prep.py}: \texttt{StatePreparationSpec} and reusable subsystem state specs.
  \item \texttt{cqed\_sim/experiment/measurement.py}: \texttt{QubitMeasurementSpec} and \texttt{measure\_qubit(...)}.
  \item \texttt{cqed\_sim/experiment/readout\_chain.py}: \texttt{ReadoutResonator}, \texttt{PurcellFilter}, \texttt{AmplifierChain}, and \texttt{ReadoutChain}.
  \item \texttt{cqed\_sim/experiment/protocol.py}: \texttt{SimulationExperiment}, \texttt{ExperimentResult}, and metadata wrappers.
\end{itemize}
These objects are not a second solver stack. They are a thin reusable layer above \texttt{SequenceCompiler} plus \texttt{simulate\_sequence(...)}. In particular:
\begin{itemize}[leftmargin=1.5em]
  \item \texttt{SimulationExperiment} is executable, not merely declarative. It resolves an initial state, compiles pulses, runs the existing solver path, and optionally performs qubit measurement on the resulting runtime state.
  \item State preparation may return either a ket or a density matrix. If any subsystem spec is mixed, the joint initial state is promoted to a density matrix before tensoring.
  \item \texttt{measure\_qubit(...)} returns exact probabilities by default. Sampled outcomes appear only when \texttt{shots} is set.
  \item When a confusion matrix is supplied, the code applies \(\mathbf{p}_{\mathrm{obs}} = M \mathbf{p}_{\mathrm{latent}}\) with columns ordered as latent \((g,e)\) and rows ordered as reported \((g,e)\).
  \item If \texttt{QubitMeasurementSpec.readout\_chain} is omitted, synthetic I/Q points are still generated from the previous Gaussian cluster model.
  \item If a \texttt{ReadoutChain} is provided, the measurement layer instead uses a resonator response, optional Purcell filter, amplifier noise, and mixer-style I/Q integration. The same API can also report \(\gamma_{\mathrm{meas}}\) and a Purcell-limited \(T_1\), and can optionally apply readout-induced dephasing plus Purcell relaxation as a reduced-state backaction model.
\end{itemize}
Direct tests: \texttt{tests/test\_26\_experimental\_protocols.py}.

\section{Physical Readout Chain}
The physical readout-chain path models a driven dispersive resonator with optional Purcell filtering and additive amplifier noise:
\begin{itemize}[leftmargin=1.5em]
  \item \texttt{ReadoutResonator} stores \(\omega_r\), \(\kappa\), the qubit-resonator coupling \(g\), the complex drive amplitude \(\epsilon\), and an optional runtime-convention dispersive pull \(\chi\).
  \item The steady-state pointer amplitudes are
  \[
  \alpha_{g/e}^{(\mathrm{ss})}
  =
  \frac{-i\epsilon}{\kappa_{\mathrm{eff}}/2 + i(\omega_{r,g/e}-\omega_d)},
  \]
  with \(\omega_{r,g}=\omega_r\) and \(\omega_{r,e}=\omega_r+\chi\) in the runtime projector convention.
  \item The implemented measurement-induced dephasing rate is
  \[
  \gamma_{\mathrm{meas}} = \frac{\kappa_{\mathrm{eff}}}{2} |\alpha_e-\alpha_g|^2,
  \]
  which reproduces the expected small-\(\chi\) scaling \(\gamma_{\mathrm{meas}} \propto \chi^2 \bar{n} / \kappa\).
  \item The Purcell filter follows the standard frequency-dependent linewidth picture:
  \[
  \kappa_{\mathrm{eff}}(\omega)
  =
  \frac{4 J^2 \kappa_f}{\kappa_f^2 + 4(\omega-\omega_f)^2},
  \]
  so the Purcell rate is evaluated as \(\Gamma_P = (g/\Delta)^2 \kappa_{\mathrm{eff}}(\omega_q)\).
  \item \texttt{AmplifierChain} applies linear gain and additive thermal noise before mixer-phase rotation and I/Q integration.
\end{itemize}
Direct tests: \texttt{tests/experiment/test\_readout\_chain.py}.

\section{Hilbert Space Structure}
\subsection*{Two-mode runtime}
\begin{itemize}[leftmargin=1.5em]
  \item Runtime ordering is qubit first, storage second:
  \[
  |q,n\rangle = |q\rangle_{\mathrm{qubit}} \otimes |n\rangle_{\mathrm{storage}}.
  \]
  \item The flattened basis index is \(\texttt{index}(q,n)=q\,n_{\mathrm{cav}}+n\), implemented in \texttt{cqed\_sim/core/conventions.py}.
  \item Computational basis is \(|g\rangle=|0\rangle\), \(|e\rangle=|1\rangle\).
\end{itemize}

\subsection*{Three-mode runtime}
\begin{itemize}[leftmargin=1.5em]
  \item The three-mode extension uses
  \[
  |q,n_s,n_r\rangle = |q\rangle_{\mathrm{qubit}} \otimes |n_s\rangle_{\mathrm{storage}} \otimes |n_r\rangle_{\mathrm{readout}}.
  \]
  \item The flattened basis index is
  \[
  \texttt{index}(q,n_s,n_r)=((q\,n_{\mathrm{storage}})+n_s)\,n_{\mathrm{readout}}+n_r.
  \]
  \item This ordering is implemented in \texttt{cqed\_sim/core/conventions.py} and \texttt{cqed\_sim/core/readout\_model.py}, and all new extractors assume subsystem \(0\) is the transmon, subsystem \(1\) is storage, subsystem \(2\) is readout.
\end{itemize}

\subsection*{Synthesis subspace}
The unitary-synthesis layer retains the same full-space qubit-first ordering, but restricted subspace bases are explicitly reordered in \texttt{cqed\_sim/unitary\_synthesis/subspace.py}; for example, a qubit-cavity block is ordered as \(|g,0\rangle, |e,0\rangle, |g,1\rangle, |e,1\rangle, \ldots\).

Direct tests: \texttt{tests/test\_25\_tensor\_product\_convention.py}, \texttt{tests/test\_27\_three\_mode\_model.py}, \texttt{tests/unitary\_synthesis/test\_subspace.py}.

\section{Operator Conventions}
\begin{itemize}[leftmargin=1.5em]
  \item The simulator uses QuTiP Pauli matrices without sign changes. Therefore
  \[
  \sigma_z |g\rangle = +|g\rangle, \qquad \sigma_z |e\rangle = -|e\rangle.
  \]
  \item The core Hamiltonians use the excitation number operator \(n_q=b^\dagger b\), not \(\sigma_z/2\). For two levels,
  \[
  n_q = |e\rangle \langle e| = \frac{I-\sigma_z}{2}.
  \]
  \item Two-mode operators in \texttt{cqed\_sim/core/model.py} are embedded as \(a = I_q \otimes a_{\mathrm{storage}}\) and \(b = b_{\mathrm{transmon}} \otimes I_s\).
  \item Three-mode operators in \texttt{cqed\_sim/core/readout\_model.py} are embedded as
  \[
  b = b_{\mathrm{transmon}} \otimes I_s \otimes I_r,\quad
  a_s = I_q \otimes a_{\mathrm{storage}} \otimes I_r,\quad
  a_r = I_q \otimes I_s \otimes a_{\mathrm{readout}}.
  \]
  \item The generic runtime now resolves drive operators through \texttt{model.drive\_coupling\_operators()}, so the solver no longer hard-codes a qubit-plus-one-mode assumption.
\end{itemize}
Direct tests: \texttt{tests/test\_16\_ideal\_primitives\_and\_extractors.py}, \texttt{tests/test\_20\_gaussian\_iq\_convention.py}, \texttt{tests/test\_25\_tensor\_product\_convention.py}, \texttt{tests/test\_27\_three\_mode\_model.py}.

\section{Hamiltonian Definitions}
\subsection*{Two-mode runtime Hamiltonian}
The reusable two-mode Hamiltonian in \texttt{cqed\_sim/core/model.py} is
\[
H_0
=
\delta_c n_c
+
\delta_q n_q
+
\frac{\alpha}{2} b^{\dagger 2} b^2
+
K(n_c)
+
\Xi(n_c) n_q,
\]
with
\[
K(n_c)
=
\frac{\texttt{kerr}}{2} n_c(n_c-1)
 + \cdots,
\qquad
\Xi(n_c)
=
\texttt{chi}\, n_c
+
\texttt{chi2}\, n_c(n_c-1)
+
\texttt{chi3}\, n_c(n_c-1)(n_c-2)
 + \cdots .
\]
\texttt{cqed\_sim/core/frequencies.py} makes the intended transition law explicit:
\[
\omega_{ge}(n)=\omega_{ge}(0)+\Xi(n).
\]

\subsection*{Textbook-to-code translation}
With the active runtime convention,
\[
n_q = \frac{I-\sigma_z}{2}.
\]
Therefore
\[
\delta_q n_q
=
\frac{\delta_q}{2} I
-
\frac{\delta_q}{2} \sigma_z,
\qquad
\Xi(n_c)n_q
=
\frac{\Xi(n_c)}{2} I
-
\frac{\Xi(n_c)}{2} \sigma_z.
\]
Up to identity and storage-only shifts,
\[
H_{q,\mathrm{runtime}} \doteq -\frac{1}{2}\bigl(\delta_q+\Xi(n_c)\bigr)\sigma_z.
\]
If one instead uses the textbook sign
\[
\sigma_z^{\mathrm{text}}=|e\rangle\langle e|-|g\rangle\langle g|=-\sigma_z,
\]
then
\[
H_{q,\mathrm{runtime}} \doteq \frac{1}{2}\bigl(\delta_q+\Xi(n_c)\bigr)\sigma_z^{\mathrm{text}}.
\]
Hence negative runtime \texttt{chi} lowers the \(|g,n\rangle \leftrightarrow |e,n\rangle\) transition frequency, and positive runtime \texttt{chi} raises it.

\subsection*{Comparison with the synthesis drift convention}
\texttt{cqed\_sim/unitary\_synthesis/sequence.py} now uses the same projector-based dispersive and Kerr semantics as the runtime model:
\[
E_{g,n}
=
\delta_c n
+
K(n),
\qquad
E_{e,n}
=
\delta_c n
+
\delta_q
+
\Xi_{\mathrm{synth}}(n)
+
K(n).
\]
Its transition shift is therefore
\[
E_{e,n}-E_{g,n} = \delta_q + \Xi_{\mathrm{synth}}(n).
\]
Matching the runtime shift \(\delta_q+\Xi_{\mathrm{runtime}}(n)\) gives
\[
\Xi_{\mathrm{synth}}(n)=\Xi_{\mathrm{runtime}}(n),
\]
so componentwise
\[
\chi_{\mathrm{synth}}=\chi_{\mathrm{runtime}},\qquad
\chi_{2,\mathrm{synth}}=\chi_{2,\mathrm{runtime}},\qquad
\chi_{3,\mathrm{synth}}=\chi_{3,\mathrm{runtime}}.
\]
Likewise the cavity self-Kerr contribution is now shared directly:
\[
K_{\mathrm{synth}}(n)=K_{\mathrm{runtime}}(n).
\]
The compatibility fields \texttt{synthesis\_chi} and \texttt{synthesis\_chi\_2} in
\texttt{cqed\_sim/analysis/parameter\_translation.py} are therefore aligned copies rather than sign-converted quantities.

\subsection*{Kerr free-evolution workflow presets}
The reusable workflow in \texttt{cqed\_sim/experiment/kerr\_free\_evolution.py} builds a two-mode
storage--transmon model from human-readable frequencies in Hz and converts them to the runtime's
angular-frequency units with \texttt{physics\_and\_conventions.conventions.to\_internal\_units()}.
Its default frame is
\[
\omega_{c,\mathrm{frame}} = \omega_c,
\qquad
\omega_{q,\mathrm{frame}} = \omega_q,
\]
so the undriven evolution isolates the nonlinear Kerr and dispersive terms after removing the bare
storage and qubit carrier rotations.

The Hamiltonian used by this workflow is therefore exactly the two-mode runtime Hamiltonian above,
evaluated in that rotating frame:
\[
\frac{H_{\mathrm{free}}}{\hbar}
=
\frac{\alpha}{2} b^{\dagger 2} b^2
+
\frac{K}{2} n_c(n_c-1)
+
\frac{K'}{6} n_c(n_c-1)(n_c-2)
+
\left[
\chi n_c
+
\chi' n_c(n_c-1)
+
\chi'' n_c(n_c-1)(n_c-2)
\right] n_q.
\]
When the qubit is initialized in \(|g\rangle\), the reduced cavity state undergoes pure self-Kerr
evolution; the dispersive coefficients remain part of the joint Hamiltonian and matter whenever a
different qubit state preparation is used.

The workflow ships with two named parameter sets:
\begin{itemize}
  \item \texttt{phase\_evolution}: \(\omega_q/2\pi = 7627.05\,\mathrm{MHz}\),
  \(\omega_c/2\pi = 8226.787\,\mathrm{MHz}\),
  \(\omega_{ro}/2\pi = 9376.75\,\mathrm{MHz}\),
  \(\alpha_q/2\pi = 283.4\,\mathrm{MHz}\),
  \(K/2\pi = -107.9\,\mathrm{kHz}\),
  \(K'/2\pi = 3.4\,\mathrm{kHz}\),
  \(\chi/2\pi = -8281.3\,\mathrm{kHz}\),
  \(\chi'/2\pi = 48.8\,\mathrm{kHz}\),
  \(\chi''/2\pi = 0.5\,\mathrm{kHz}\).
  \item \texttt{value\_2}: same \(\omega_q\), \(\omega_c\), \(\omega_{ro}\), and \(\alpha_q\), with
  \(K/2\pi = -106.2\,\mathrm{kHz}\),
  \(K'/2\pi = 3.4\,\mathrm{kHz}\),
  \(\chi/2\pi = -8273\,\mathrm{kHz}\),
  \(\chi'/2\pi = 60.9\,\mathrm{kHz}\), and \(\chi''\) taken as zero because no alternate value was supplied.
\end{itemize}

The readout frequency \(\omega_{ro}\) is stored with the preset metadata for consistency with the
measured parameter set, but it is not part of the two-mode Kerr free-evolution Hamiltonian.

\subsection*{Three-mode storage-transmon-readout Hamiltonian}
The three-mode model in \texttt{cqed\_sim/core/readout\_model.py} implements
\[
\frac{H}{\hbar}
=
\delta_s n_s
+
\delta_r n_r
+
\delta_q n_q
+
\frac{\alpha}{2} b^{\dagger 2} b^2
+
\chi_s n_s n_q
+
\chi_r n_r n_q
+
\chi_{sr} n_s n_r
+
\frac{K_s}{2} n_s(n_s-1)
+
\frac{K_r}{2} n_r(n_r-1).
\]
This is intentionally kept in the same runtime convention style as the two-mode path:
\begin{itemize}[leftmargin=1.5em]
  \item \(n_q=b^\dagger b\) is the transmon excitation projector/number operator.
  \item Positive \(\chi_s\) or \(\chi_r\) raises the transmon transition frequency as storage or readout occupancy increases.
  \item Positive \(\chi_{sr}\) raises the storage frequency with readout occupancy and raises the readout frequency with storage occupancy.
  \item Positive \(K_s\) or \(K_r\) raises the adjacent-mode spacing by \(+K n\).
\end{itemize}
The model also exposes analytic helpers for the basis energies and the qubit, storage, and readout transition frequencies.

\subsection*{Optional nonlinear and exchange terms}
\texttt{cqed\_sim/sim/couplings.py} now provides explicit builder functions for
\[
\chi_{ab} a^\dagger a\, b^\dagger b,\qquad
+\frac{K}{2} a^{\dagger 2} a^2,\qquad
J(a^\dagger b + a b^\dagger),
\]
and the model classes accept \texttt{CrossKerrSpec}, \texttt{SelfKerrSpec}, and \texttt{ExchangeSpec} entries that are injected into the static Hamiltonian assembly. This extends the existing dispersive models without replacing their original API.

\subsection*{Standard-compatible versus repo-specific}
\begin{itemize}[leftmargin=1.5em]
  \item Standard-compatible: qubit-first tensor ordering, QuTiP Pauli signs, rad/s and seconds internally, Duffing/Kerr terms, Lindblad dynamics, and the canonical rule that negative \(\chi\) lowers the qubit transition with photon number while positive \(\chi\) raises it.
  \item Repo-specific: writing the transmon term in terms of \(n_q\), storing higher-order pulls and Kerr terms in falling-factorial form, and using complex envelopes with \(e^{+i(\omega t+\phi)}\).
  \item Needs caution: raw \texttt{Pulse.carrier} remains the negative of the rotating-frame transition frequency, so user-facing spectroscopy axes should use \texttt{carrier\_for\_transition\_frequency(...)} or \texttt{transition\_frequency\_from\_carrier(...)} rather than raw carriers.
\end{itemize}

\section{Frequency and Time Units}
\begin{itemize}[leftmargin=1.5em]
  \item Internal Hamiltonian coefficients and rotating-frame frequencies are in rad/s.
  \item Pulse durations, timeline steps, and solver \texttt{max\_step} values are in seconds.
  \item The repository now centralizes conversion helpers in \texttt{physics\_and\_conventions/conventions.py}, including \texttt{to\_internal\_units(...)} and \texttt{from\_internal\_units(...)}.
  \item Example-side configuration files may still carry \texttt{\_hz}, \texttt{\_ns}, or \texttt{\_1\_per\_s} field names, but those values are converted before constructing library models or noise objects.
\end{itemize}
Direct tests: \texttt{tests/test\_05\_detuning\_and\_frames.py}, \texttt{tests/test\_10\_chi\_convention.py}, \texttt{tests/test\_13\_hardware\_extended.py}, \texttt{tests/test\_27\_three\_mode\_model.py}.

\section{Parameter Translation, Calibration Targets, and Backends}
\begin{itemize}[leftmargin=1.5em]
  \item \texttt{cqed\_sim/analysis/parameter\_translation.py} now provides \texttt{from\_transmon\_params(...)} and \texttt{from\_measured(...)}. Bare transmon parameters use the usual large-\(E_J/E_C\) expansion \(\omega_{01}\approx\sqrt{8E_JE_C}-E_C\) and \(\alpha\approx-E_C\), while dressed \(\chi\) and \(\chi_2\) are extracted by matching a low-dimensional Duffing-plus-resonator diagonalization to the runtime form \(\omega_{ge}(n)=\omega_q+\chi n+\chi_2 n(n-1)\).
  \item \texttt{cqed\_sim/calibration\_targets/} now provides reusable spectroscopy, Rabi, Ramsey, \(T_1\), \(T_{2,\mathrm{echo}}\), and DRAG-tuning surrogate targets that return fitted parameters, uncertainties, and raw simulated data.
  \item \texttt{cqed\_sim/backends/} now provides \texttt{NumPyBackend} and \texttt{JaxBackend} plus a dense piecewise-constant solver path in \texttt{cqed\_sim/sim/solver.py}. The default runtime still uses QuTiP; the backend path is opt-in through \texttt{SimulationConfig.backend}.
\end{itemize}
Direct tests: \texttt{tests/analysis/test\_parameter\_translation.py}, \texttt{tests/calibration\_targets/test\_targets.py}, \texttt{tests/sim/test\_backends.py}.

\section{Rotating Frame Conventions}
\begin{itemize}[leftmargin=1.5em]
  \item Frames are represented by \texttt{FrameSpec(omega\_c\_frame, omega\_q\_frame, omega\_r\_frame)} in \texttt{cqed\_sim/core/frame.py}.
  \item The legacy field \texttt{omega\_c\_frame} remains the storage/cavity frame frequency. The readout model adds the alias property \texttt{omega\_s\_frame} for clarity.
  \item The two-mode path uses \(\delta_c=\omega_c-\omega_c^{\mathrm{frame}}\) and \(\delta_q=\omega_q-\omega_q^{\mathrm{frame}}\).
  \item The three-mode path uses \(\delta_s=\omega_s-\omega_s^{\mathrm{frame}}\), \(\delta_r=\omega_r-\omega_r^{\mathrm{frame}}\), and \(\delta_q=\omega_q-\omega_q^{\mathrm{frame}}\).
  \item \texttt{manifold\_transition\_frequency(...)} still refers only to the two-mode path; the three-mode model instead exposes \texttt{qubit\_transition\_frequency(...)}, \texttt{storage\_transition\_frequency(...)}, and \texttt{readout\_transition\_frequency(...)}.
\end{itemize}
Direct tests: \texttt{tests/test\_05\_detuning\_and\_frames.py}, \texttt{tests/test\_21\_qubox\_convention\_reconciliation.py}, \texttt{tests/test\_27\_three\_mode\_model.py}.

\section{Drive Hamiltonians}
\begin{itemize}[leftmargin=1.5em]
  \item \texttt{cqed\_sim/pulses/pulse.py} samples analytic or discrete envelopes as
  \[
  \epsilon(t)=\texttt{amp}\,\texttt{envelope}(t)\,e^{i(\texttt{carrier} t+\texttt{phase})}.
  \]
  \item \texttt{cqed\_sim/sim/runner.py} inserts \(\epsilon(t)\) and \(\epsilon^*(t)\) as raising and lowering couplings obtained from the model, rather than from a hard-coded target list.
  \item Because the runtime drive term is \(\epsilon(t)\,b^\dagger + \epsilon^*(t)\,b\), a rotating-frame transition at frequency \(\omega_{\mathrm{transition}}\) is resonantly addressed by \(\texttt{carrier}=-\omega_{\mathrm{transition}}\). The public helpers \texttt{carrier\_for\_transition\_frequency(...)} and \texttt{transition\_frequency\_from\_carrier(...)} make that mapping explicit.
  \item User-facing spectroscopy examples and the updated \texttt{usage\_examples.ipynb} therefore plot physical transition detuning on the x-axis and convert that value to the internal raw carrier with \texttt{carrier\_for\_transition\_frequency(...)} before simulation.
  \item The two-mode path supports \texttt{"cavity"} or \texttt{"storage"}, \texttt{"qubit"}, and the legacy \texttt{"sideband"} target, plus the structured targets \texttt{TransmonTransitionDriveSpec(lower\_level, upper\_level)} and \texttt{SidebandDriveSpec(mode, lower\_level, upper\_level, sideband)}.
  \item The three-mode path supports \texttt{"storage"}, \texttt{"readout"}, \texttt{"qubit"}, and \texttt{"transmon"} targets, plus the same structured transition and sideband targets. The readout and storage drives use the same complex-envelope sign convention as the existing qubit and storage drives.
  \item The effective three-mode drive Hamiltonian is
  \[
  H_d / \hbar
  =
  \epsilon_s(t) a_s^\dagger + \epsilon_s^*(t) a_s
  +
  \epsilon_q(t) b^\dagger + \epsilon_q^*(t) b
  +
  \epsilon_r(t) a_r^\dagger + \epsilon_r^*(t) a_r .
  \]
\end{itemize}
Direct tests: \texttt{tests/test\_20\_gaussian\_iq\_convention.py}, \texttt{tests/test\_23\_sqr\_additive\_amplitude\_correction.py}, \texttt{tests/test\_28\_three\_mode\_drives\_and\_noise.py}, \texttt{tests/test\_31\_multilevel\_sideband\_extension.py}.

\subsection*{Effective multilevel sideband model}
The reusable sideband path added in \texttt{cqed\_sim/core/drive\_targets.py} and
\texttt{cqed\_sim/pulses/builders.py} is an effective rotating-wave Hamiltonian, not a
microscopic flux-drive derivation. For a red sideband on a bosonic mode \(a_m\) and a
selected transmon transition \(|\ell\rangle \leftrightarrow |u\rangle\), the implemented
operator pair is
\[
H_{\mathrm{sb}}/\hbar
=
\epsilon(t)\, |u\rangle\langle \ell|\, a_m
+
\epsilon^*(t)\, a_m^\dagger |\ell\rangle\langle u|,
\]
with an analogous blue-sideband variant that swaps \(a_m\) and \(a_m^\dagger\). The public
\texttt{SidebandDriveSpec} selects the bosonic mode, the ancilla ladder transition, and the
red/blue branch. The helper \texttt{effective\_sideband\_rabi\_frequency(g, \(\delta\))}
uses the standard effective-model rule
\[
\Omega_{\mathrm{eff}} = \sqrt{(2g)^2 + \delta^2}.
\]
This path is intended for fast-control and synchronization studies where the effective
manifold dynamics are the quantity of interest.

\subsection*{Kerr-sign notebook diagnosis}
The paper-comparison notebook \texttt{test\_against\_papers/bosonic\_controls.ipynb} was
audited against the runtime implementation. The core simulator and the documented physics
convention both use
\[
H_K/\hbar = +\frac{K}{2} a^{\dagger 2} a^2.
\]
For the measured parameter set used in the notebook, the runtime evolution matches the
documented negative-\(K\) behavior. The apparent sign flip comes from interpreting the
rotating-frame coherent-state phase trajectory, not from a flipped self-Kerr term in the
Hamiltonian builder. This is now guarded by \texttt{tests/test\_32\_kerr\_sign\_notebook\_regression.py}
and exposed through \texttt{cqed\_sim.experiment.verify\_kerr\_sign(...)}.

\section{Dissipation and Lindblad Terms}
\begin{itemize}[leftmargin=1.5em]
  \item \texttt{NoiseSpec} in \texttt{cqed\_sim/sim/noise.py} now supports \texttt{t1}, \texttt{transmon\_t1}, \texttt{tphi}, \texttt{kappa}, \texttt{nth}, \texttt{kappa\_storage}, \texttt{kappa\_readout}, \texttt{nth\_storage}, and \texttt{nth\_readout}.
  \item The implemented convention is \(\gamma_1 = 1/T_1\) and \(\gamma_\phi = 1/(2 T_\phi)\).
  \item If \texttt{transmon\_t1=(T_{1,ge}, T_{1,fe}, \ldots)} is supplied, the collapse-operator builder emits explicit ladder decays \(\sqrt{1/T_{1,j}}\,|j-1\rangle\langle j|\). If it is not supplied, the older aggregate \(\sqrt{\gamma_1}\,b\) path remains available through \texttt{t1}.
  \item Two-level dephasing uses \(\sqrt{\gamma_\phi}\,\sigma_z\) with \(\sigma_z = I - 2 n_q\). Multi-level transmon dephasing uses \(\sqrt{\gamma_\phi}\,n_q\).
  \item In the two-mode path, \texttt{kappa} and \texttt{nth} still drive storage loss and thermal excitation.
  \item In the three-mode path, storage and readout can decay independently via \texttt{kappa\_storage} and \texttt{kappa\_readout}, with separate thermal occupations.
  \item \texttt{simulate\_sequence(...)} uses \texttt{mesolve} when collapse operators are present or when the initial state is already a density matrix; otherwise it uses \texttt{sesolve}.
\end{itemize}
Direct tests: \texttt{tests/test\_14\_dissipation.py}, \texttt{tests/test\_18\_noise\_and\_sideband.py}, \texttt{tests/test\_28\_three\_mode\_drives\_and\_noise.py}, \texttt{tests/test\_31\_multilevel\_sideband\_extension.py}.

\section{State Representation}
\begin{itemize}[leftmargin=1.5em]
  \item Runtime inputs may be kets or density matrices.
  \item The result wrapper stores the final state, optional state history, expectation traces, and the raw QuTiP solver result.
  \item The generic extractor path now supports two-mode and three-mode states through subsystem-aware partial traces.
  \item \texttt{reduced\_cavity\_state(...)} remains specific to the two-mode path. The three-mode path uses \texttt{reduced\_storage\_state(...)} and \texttt{reduced\_readout\_state(...)}.
  \item \texttt{measure\_qubit(...)} acts on the reduced transmon state obtained from the runtime state convention. The exact probabilities are taken from the reduced density matrix before any optional confusion-matrix application or finite-shot sampling.
  \item Synthetic I/Q points remain available for lightweight sampling-only usage.
  \item The new readout-chain path adds optional state-conditioned resonator traces, amplifier-noise I/Q sampling, and a reduced-state backaction model without changing the default no-chain behavior.
\end{itemize}

\section{Hilbert Space Truncation}
\begin{itemize}[leftmargin=1.5em]
  \item The two-mode model uses \texttt{n\_cav} for storage truncation and \texttt{n\_tr} for transmon truncation.
  \item The three-mode model uses \texttt{n\_storage}, \texttt{n\_readout}, and \texttt{n\_tr}.
  \item The multilevel sideband and structured-transition helpers assume the selected ancilla levels lie within the chosen transmon truncation and will raise if the requested transition falls outside \([0, \texttt{n\_tr}-1]\).
  \item State preparation helpers pad or truncate explicit amplitude vectors to the requested subsystem dimension.
  \item SQR angle arrays are still padded to the storage dimension in \texttt{cqed\_sim/pulses/calibration.py}.
  \item The synthesis layer still distinguishes the full Hilbert dimension from restricted subspace dimension in \texttt{cqed\_sim/unitary\_synthesis/subspace.py}.
\end{itemize}

\section{Gaps Between Code Conventions and Test Coverage}
\begin{itemize}[leftmargin=1.5em]
  \item The new runtime-to-synthesis translation helper exposes \(\chi_{\mathrm{synth}}=-\chi_{\mathrm{runtime}}/2\), but the library still does not force every synthesis-side caller to use that mapping automatically.
  \item Higher-order \(\chi_2\), \(\chi_3\), and higher-order Kerr coefficients are covered indirectly, but not with isolated sign-and-factor analytic tests of the same style now used for the three-mode diagonal-energy checks.
  \item The readout-chain model is physically motivated, but it is still an effective resonator/filter/amplifier abstraction rather than a full microwave network or POVM calibration workflow.
  \item The backend path is dense and piecewise constant; it is intended for parity and small-system acceleration checks, not as a drop-in replacement for every QuTiP workflow.
\end{itemize}

\section{Potential Ambiguities}
\textbf{Moderate:} the repository migrated on March 11, 2026 from an older \(-\chi n_c n_q\) convention to the current \(+\chi n_c n_q\) convention. Historical notes or saved plots created before that migration may still use the old sign.\\
\textbf{Moderate:} higher-order pull and Kerr parameters are falling-factorial coefficients, not ordinary polynomial coefficients.\\
\textbf{Moderate:} the runtime writes transmon physics in terms of \(n_q\) rather than \(\sigma_z/2\), so ground-state energy offsets differ from some textbook presentations.\\
\textbf{Moderate:} synthetic I/Q output is a convenience sampling layer, not a calibrated hardware-response model.\\
\textbf{Low:} the explicit complex-envelope sign \(e^{+i(\omega t+\phi)}\) is repo-specific and must be preserved when adding pulse families.\\
\textbf{Low:} \texttt{FrameSpec} still stores the storage frame in the legacy field \texttt{omega\_c\_frame}; the \texttt{omega\_s\_frame} alias removes ambiguity but does not rename the underlying field.

\section{Summary}
\texttt{cqed\_sim} is now a coherent reusable library for pulse-level cQED simulation with both a two-mode transmon-storage path and a three-mode storage-transmon-readout extension. The authoritative runtime conventions are: qubit-first tensor ordering; \(|g\rangle=|0\rangle\), \(|e\rangle=|1\rangle\); QuTiP-standard Pauli signs; rad/s internally and seconds for time; complex envelopes with \(e^{+i(\omega t+\phi)}\) and therefore \(\texttt{carrier}=-\omega_{\mathrm{transition}}\) for resonant rotating-frame spectroscopy; dispersive physics written in terms of \(n_q=b^\dagger b\) with \(+\chi n_c n_q\)-style conditional terms; and Lindblad collapse operators as implemented in \texttt{cqed\_sim/sim/noise.py}. Negative \(\chi\) now means that higher cavity photon number lowers the qubit transition frequency, and positive Kerr coefficients raise adjacent bosonic transition spacings. The experiment-facing layer now has two measurement modes: a default lightweight exact-probability path and an opt-in physical readout-chain path with resonator response, Purcell filtering, amplifier noise, and reduced-state backaction. The library also exposes optional nonlinear/exchange Hamiltonian terms, aligned runtime/synthesis parameter translation helpers, calibration-target utilities, and an opt-in dense NumPy/JAX backend path.
